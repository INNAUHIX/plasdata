# 认证证书弹窗图片样式修改计划

## 问题描述
将不支持自助下载的认证证书弹窗改成图片样式，要求完全还原图片的结构和UI，并且弹窗的证书信息要与外面点击的认证证书保持一致。

## 现有代码分析

当前 `CertificateDownloadModal` 组件在处理不支持自助下载的情况时，使用的是一个简单的提示框：

```tsx
{needService ? (
  <div className="rounded-xl bg-[#fff7ed] px-4 py-4 text-[14px] text-[#9a3412]">
    该证书资源暂不支持自助下载，请联系客服获取（热线：18823327240，微信同号）。
  </div>
) : null}
```

## 目标样式分析

根据用户提供的图片，目标样式应该包含以下元素：

1. **标题**：证书资源下载
2. **左侧区域**：
   - 证书标志（如FDA标志）
   - 证书名称（如美国FDA认证）
   - 更新时间
   - 有效时间
   - 非公开资料，请咨询客服获取（橙色文字）
3. **右侧区域**：
   - 二维码
   - 添加客服微信号
   - 联系客服获取
4. **关闭按钮**：右上角的关闭按钮

## 修改方案

### 1. 调整整体布局
- 当 `needService` 为 true 时，使用 `flex` 布局替代当前的简单提示框
- 左侧和右侧区域使用 `flex-1` 来平均分配空间

### 2. 实现左侧区域
- 使用传入的 `certName` 作为证书名称
- 使用传入的 `revisionDate` 作为更新时间
- 显示有效时间（如果有相关数据）
- 创建证书标志的灰色占位符
- 显示橙色的提示文字

### 3. 实现右侧区域
- 创建二维码的灰色占位符
- 显示 "添加客服微信号" 和 "联系客服获取" 文字

### 4. 保持关闭按钮位置
- 保持关闭按钮在右上角的位置不变

### 5. 移除不需要的元素
- 当 `needService` 为 true 时，移除收藏、分享和预览按钮
- 移除底部的 "联系客服" 按钮，因为这些信息已经在右侧区域显示

## 实施步骤

1. 打开 `src/components/CertificateDownloadModal.tsx` 文件
2. 修改 `needService` 条件下的渲染内容，替换为图片样式的布局
3. 实现左侧和右侧区域的布局和样式
4. 使用组件接收到的 `certName` 和 `revisionDate` 等 props 来显示证书信息
5. 添加灰色占位符替代无法还原的图片
6. 调整各个元素的位置和样式，确保与图片一致
7. 测试修改效果
8. 自我检查，确保与图片的结构和UI一致，并且证书信息与外面点击的认证证书保持一致

## 预期结果

- 不支持自助下载的认证证书弹窗显示为图片样式
- 左侧显示证书标志和认证信息，且信息与外面点击的认证证书一致
- 右侧显示二维码和联系信息
- 各个元素的位置与图片一致
- 部分无法还原的图片使用灰色占位符替代
